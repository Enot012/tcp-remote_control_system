# TCP Server Manager

Административный TCP-сервер с графическим интерфейсом на tkinter.
Позволяет управлять подключёнными клиентами: выполнять команды, передавать файлы,
настраивать группы и планировать задачи — всё через GUI без единого `input()`.

---

## Архитектура

```
gui_4.py  ──запускает──▶  server.py  (asyncio, отдельный поток)
               │                │
               │         handlers.py  (бизнес-логика команд)
               │                │
               └──────────▶  managers.py  (состояние, пользователи, файлы, логи)
                                │
                           config.py  (константы, enum, парсеры)
```

**Два потока выполнения:**

| Поток | Что живёт |
|---|---|
| Главный (tkinter) | `ServerApp`, все диалоги, цикл mainloop |
| Фоновый (asyncio) | TCP-сервер, обработка клиентов, периодические задачи |

Команды из GUI передаются в asyncio через `asyncio.run_coroutine_threadsafe()`.
Логи из asyncio-потока попадают в GUI через callback `set_log_callback()`.

## Файловая структура (рабочая директория `home/`)

```
home/
├── code.txt                     # Список команд для SIMPL
├── users.json                   # База пользователей
├── server_state.json            # Снимок состояния сервера
├── crash.log                    # Лог критических ошибок
├── save/                        # Сохранённые выводы команд (save)
├── trash/                       # Авто-сохранение всех выводов (CommandMonitor)
├── history/                     # История сессий по пользователям
├── files/
│   └── scheduled_commands/      # Результаты отложенных команд
├── logs/                        # Ежедневные лог-файлы (YYYY-MM-DD.log)
├── json/
│   ├── groups.json              # Группы пользователей
│   └── scheduled_commands.json  # Очередь отложенных команд
└── send_file/                   # Файлы, готовые к отправке клиентам (IMPORT)
```

## Команды

### Онлайн (выполняются немедленно для подключённых клиентов)

| Команда | Синтаксис | Описание |
|---|---|---|
| `cmd` | `cmd <user\|all\|group:X> <команда>` | Выполнить shell-команду на клиенте |
| `simpl` | `simpl <user\|all\|group:X>` | Отправить все команды из `code.txt` |
| `export` | `export <user> <путь_клиент> [путь_сервер]` | Получить файл(ы) с клиента на сервер |
| `import` | `import <user> <путь_сервер> [путь_клиент]` | Отправить файл(ы) с сервера клиенту |
| `save` | `save <user\|all> <filename>` | Сохранить последний вывод в файл |
| `cancel` | `cancel <user>` | Отменить выполняющуюся команду |
| `kick` | `kick <user\|all>` | Принудительно отключить клиента |
| `rename` | `rename <user> <alias>` | Переименовать пользователя |
| `list` | `list` | Показать всех пользователей |
| `status` | `status` | Показать активные команды |

### Группы

| Команда | Синтаксис |
|---|---|
| `group_new` | `group_new <name> [user1 user2 ...]` |
| `group_list` | `group_list` |
| `group_del` | `group_del <name>` |
| `group_add` | `group_add <group> <user1> [user2 ...]` |
| `group_rm` | `group_rm <group> <user1> [user2 ...]` |

### Отложенные команды (выполняются при следующем подключении клиента)

| Команда | Описание |
|---|---|
| `chart_new` | Создать отложенную задачу (CMD / SIMPL / IMPORT / EXPORT) |
| `chart_list` | Список задач в очереди |
| `chart_del` | Удалить задачу по индексу |
| `chart_comd` | Показать задачи с деталями |

### Системные

| Команда | Описание |
|---|---|
| `help` | Показать список команд |
| `exit` | Завершить сервер |

## Подстановки путей

В аргументах команд поддерживаются переменные:

| Переменная | Значение |
|---|---|
| `{user}` | Имя (alias) целевого пользователя |
| `{path}` | `default_path` пользователя из `users.json` |
| `{path_serv}` | Абсолютный путь к директории `home/send_file/` |

## Клиентский протокол (TCP)

**Подключение:** клиент отправляет одну строку: `username,os_name,home_path\n`

**Сообщения от клиента → серверу:**

| Сообщение | Описание |
|---|---|
| `output:start:` | Начало вывода команды |
| `output:chunk:<данные>` | Фрагмент вывода (перенос строки = `<<<NL>>>`) |
| `output:end` | Конец вывода |
| `filetru:start:` / `filetru:chunk:` / `filetru:end` | То же для SIMPL (multiple commands) |
| `export:start:<json>` | Клиент начинает передачу файлов |
| `import:complete` | Клиент успешно принял файл |
| `import:error:<msg>` | Ошибка при приёме файла |

**Сообщения от сервера → клиенту:**

| Сообщение | Описание |
|---|---|
| `CMD:<команда>` | Выполнить shell-команду |
| `FILETRU:<команда>` | Выполнить команду (режим SIMPL) |
| `EXPORT;<src>;<dst>` | Отправить файл(ы) серверу |
| `IMPORT:START:<json>` + `FILE:META:<json>` | Принять файл(ы) от сервера |
| `KICK:<причина>` | Принудительное отключение |

## Настройка (`config.py`)

```python
HOST = "0.0.0.0"       # Адрес прослушивания
PORT = 9000            # Порт

COMMAND_TIMEOUT  = 120  # Таймаут команды (сек) — после него CANCEL
WARNING_TIMEOUT  = 90   # Предупреждение клиенту до таймаута
READ_TIMEOUT     = 300  # Таймаут чтения TCP
STATE_SAVE_INTERVAL = 30  # Интервал автосохранения состояния
TIMEZONE_OFFSET  = +1   # Смещение от UTC для логов
CHUNK_SIZE       = 65536  # Размер чанка при передаче файлов
```

## Запуск

```bash
# С GUI
python gui.py

# Без GUI (терминал)
python server.py
```

**Зависимости:** стандартная библиотека Python 3.11+ (`asyncio`, `tkinter`, `json`, `pathlib`).
Сторонних пакетов нет.

## Описание файлов

| Файл | Назначение                                                                         |
|---|------------------------------------------------------------------------------------|
| `config.py` | Константы, enum-константы, парсеры входных строк, цветовая схема GUI               |
| `managers.py` | Состояние сервера, пользователи, группы, отложенные команды, логи, передача файлов |
| `handlers.py` | Бизнес-логика всех команд; диспетчеры серверных и клиентских сообщений             |
| `server.py` | asyncio TCP-сервер; точка входа для фонового потока                                |
| `gui_4.py` | tkinter GUI: главное окно, панели, диалоги                                         |
